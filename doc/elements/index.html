Elements: A DSL for optimizing and calculating Genshin Impact damage rotations🔗ℹ

Joshua Goldberg <goldberg.josh@northeastern.edu>
Will Du <du.wi@northeastern.edu>

 (require elements) package: elements
Genshin Impact is an open world RPG released in 2020. In Genshin, you control a team of up to four characters, and engage in various levels of combat. The goal is typically to clear content as fast as possible, which requires efficient team rotations with high levels of sustained damage. This package implements a damage calculator as a hosted DSL. With Elements, you can easily define the basics of a team lineup, including weapons, elemental skills, and characters. From there, you can setup damage rotations against enemies of varying toughness, and calculate potential damage output. Additionally, Elements compares old rotations to newer ones, allowing you to easily keep track of what combination has provided the highest sustained damage.

syntax

(genshin-calc expr ...)

Used to compile every expr within it. Required for proper compile-time error checking.

syntax

(define-weapon name atk attr buffs ...)

 
  name : identifier?
  atk : number?
  attr : modifier-attribute?
  buffs : buff?
Defines a weapon with a base attack number that buffs a characters stats based on the given modifier attribute attr. The weapon also has a list of buffs.

> (define-weapon wolfs-gravestone
    550
    (mod critr 24.1)
    (triggered-buff
     [dmgup
      #:effect (buff atk% 20.0)
      #:trigger normal-attack
      #:limit 1
      #:party-wide #f
      #:duration 10.0])
    (unconditional-buff
     [crit-up
      #:effect (buff critd 20)
      #:party-wide #f]))

syntax

(define-skill name
  #:cooldown cd
  #:attr attr
  #:duration duration
  #:type type
  buffs ...)
 
  name : identifier?
  cd : number?
  attr : dmg-attribute?
  duration : number?
  type : element?
  buffs : buff?
Defines a character skill/burst that has a cooldown, duration, and a list of buffs. The skill also deals damage based on the given damage attribute attr.

> (define-skill all-attack-up
    #:cooldown 25.0
    #:attr (base-dmg atk% 125)
    #:duration 0.1
    #:type pyro
    (applied-buff
     [skill-atkup
      #:effect (buff atk 125)
      #:limit 1
      #:party-wide #t
      #:duration 10]))
> (define-skill basic-slash
    #:cooldown 5.0
    #:attr (base-dmg atk% 25)
    #:duration 1.0
    #:type pyro
    (applied-buff
     [skill-hpup
      #:effect (buff hp 20)
      #:limit 1
      #:party-wide #f
      #:duration 10.0]))

syntax

(define-attack-sequence name
  ([attr duration type] ...
   #:charged  [attr2 duration2 type2]
   #:plunging [attr3 duration3 type3]))
 
  name : identifier?
  attr : dmg-attribute?
  duration : number?
  type : element?
  attr2 : dmg-attribute?
  duration2 : number?
  type2 : element?
  attr3 : dmg-attribute?
  duration3 : number?
  type3 : element?
Defines a character’s attack sequence. Each attack has its own unique duration and an element type. Attacks deal damage based on the given damage attribute attr. An attack sequence also includes a charged and plunging attack, with their own corresponding duration, damage, and element type.

> (define-attack-sequence attack-chain
    ([(base-dmg atk% 10) 0.5 physical]
     [(base-dmg atk% 25) 0.2 physical]
     [(base-dmg atk% 125) 0.8 physical]
     [(base-dmg atk% 250) 1.5 physical]
     #:charged [(base-dmg hp% 5) 3.5 pyro]
     #:plunging [(base-dmg hp% 10) 3.5 physical]))

syntax

(define-artifact name
  set-name
  main-attr
  sub-attr ...)
 
  name : identifier?
  set-name : string?
  main-attr : modifier-attribute?
  sub-attr : modifier-attribute?
Defines an artifact piece. Artifacts belong to a set, specified by the string provided by set-name. Each artifact also augments a character’s stats with a modifier attribute main-attr, as well as a list of more modifier attribute sub attributes.

> (define-artifact test-feather
    "cool feather collection"
    (mod atk 325)
    (mod atk 27)
    (mod em 42))
> (define-artifact test-goblet
    "cool goblet collection"
    (mod critr 46.6)
    (mod critd 16.2)
    (mod critr 3.0)
    (mod def 128))

> (define-team-lineup lone-member (test-char))

syntax

(calculate-rotation-damage lineup enemy (attk ...))

 
  lineup : identifier?
  enemy : identifier?
  attk : attack-key?
Calculates and displays the amount of damage done by a team lineup against a specified enemy, given a list of attack keys specifying character actions. Also compares the result to previous entries with the same lineup and enemy, and displays the best saved calculation.

> (calculate-rotation-damage lone-member dummy (N C C C N N N N N N ND C))

[]=======================================================================================[]

  Simulation run with input string: (N C C C N N N N N N ND C)

  Total damage: 20162.73 Total time: 18.3 seconds DPS: 1101.79

  The best run with this layout was a sequence of: (N C C C N N N N N N ND C)

  Total damage: 20162.73 Total time 18.3 seconds DPS: 1101.79

[]=======================================================================================[]

> (calculate-rotation-damage two-members dummy (E Q N N N N (Swap 1) N N N ND))

[]=======================================================================================[]

  Simulation run with input string: (E Q N N N N (Swap 1) N N N ND)

  Total damage: 29698.72 Total time: 5.7 seconds DPS: 5210.3

  The best run with this layout was a sequence of: (E Q N N N N (Swap 1) N N N ND)

  Total damage: 29698.72 Total time 5.7 seconds DPS: 5210.3

[]=======================================================================================[]

'(29698.716276460007 5.699999999999999)

1 Buffs🔗ℹ

A buff augments characters when active, changing the values of a character’s stats by a specified amount. A buff can either be a unconditional-buff, or a trigger-buff.

buff syntax

(unconditional-buff [name #:effect attr
                     #:party-wide party-wide])
 
  attr : buff-attribute?
  party-wide : boolean?
An unconditional buff that always applies the attribute given by attr. If party-wide is #t, this buff will also apply to all characters in the lineup, not just the character the buff is linked to.

1.1 Trigger buffs🔗ℹ

A trigger buff augments characters, but unlike conditionals, is not always active. A trigger buff can be either a triggered-buff or an applied-buff.

trigger buff syntax

(triggered-buff [name #:effect attr
                 #:trigger t
                 #:limit limit
                 #:party-wide party-wide
                 #:duration duration])
 
  attr : buff-attribute?
  t : trigger?
  limit : integer?
  party-wide : boolean?
  duration : number?
A triggered buff that applies the attribute given by attr, when a trigger t is performed by the character. The buff lasts for the specified time given by duration before going inactive. Each triggered buff can stack up until the limit specified by limit, at which point any more activations of the buff will refresh the it at the max limit. If party-wide is #t, this buff will also apply to all characters in the lineup, not just the character the buff is linked to.

trigger buff syntax

(applied-buff [name #:effect attr
               #:limit limit
               #:party-wide party-wide
               #:duration duration])
 
  attr : buff-attribute?
  limit : integer?
  party-wide : boolean?
  duration : number?
An applied buff that operates similarly to a triggered-buff, except without a specific trigger. Once activated, an applied buff applies the attribute given by attr. The buff lasts for the specified time given by duration before going inactive. Each triggered buff can stack up until the limit specified by limit, at which point any more activations of the buff will refresh the it at the max limit. If party-wide is #t, this buff will also apply to all characters in the lineup, not just the character the buff is linked to.

2 Stats🔗ℹ

Every character has a set of stats, which can either be modified, or be used to calculate and modify damage. Additionally, characters can equip a weapon and multiple artifacts to further augment that stats they already have.

  stat = scaling-stat
  | flat-stat
  | percent-stat
     
  char-stat = base-stat
  | crit-stat
     
  scaling-stat = base-stat
  | dmg
     
  base-stat = hp
  | atk
  | def
  | em
     
  flat-stat = crit-stat
  | dmg%
     
  crit-stat = critr
  | critd
     
  percent-stat = hp%
  | atk%
  | def%
  | em%

All stats can be increased by a flat value.

2.1 Char stats🔗ℹ

A char stat is one of the six stats a character can have.

2.2 Scaling stats🔗ℹ

A scaling stat is a stat that can scale off of a percent stat.

2.3 Base stats🔗ℹ

A base stat is one of a character’s main stat attributes.

They have corresponding percent stats that scaling stats scale off of.

2.4 Flat stats🔗ℹ

A flat stat is a stat that can only increase by a flat value. They are either a crit stat or "dmg%"

3 Attributes🔗ℹ

An attribute is a calculation that depends on one or more stats. There are multiple variations depending on the action that attribute is being used to perform.

3.1 Buff attributes🔗ℹ

buff attribute syntax

(buff attr value)

(buff scaling-attr (scale-attr value))
 
  attr : stat?
  scaling-attr : scaling-stat?
  scale-attr : percent-stat?
  value : number?
A buff attribute indicates how a buff augments the stats of a character, and can be written in multiple ways.

In the first usage, increases a base stat attr by value. If attr is a percent stat, then it increases the stat by a percentage equal to (value/100).

In the second usage, increases a base stat base-attr by (sattr * (value/100)), where sattr is a percent stat.

3.2 Modifier attributes🔗ℹ

modifier attribute syntax

(mod char-attr value)

(mod percent-attr value)
 
  char-attr : char-stat?
  percent-attr : percent-stat?
  value : number?
A modifier attribute indicates how to augment the stats of a character. However, it is more restrictive compared to a buff attribute, as it does not allow stat increases to be dependent on other stats and does not allow "dmg%" increases.

3.3 Damage attributes🔗ℹ

damage attribute syntax

(base-dmg percent-attr value)

 
  percent-attr : percent-stat?
  value : number?
A damage attribute indicates what percent-stat to scale an instance of damage off of by a percentage equal to (value/100).
